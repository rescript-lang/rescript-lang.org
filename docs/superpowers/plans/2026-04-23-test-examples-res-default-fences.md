# Test Examples Res Default Fences Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make plain ` ```res ` fences checked by default, add ` ```res nocheck ` as the explicit opt-out, expand ReScript-first `CodeTab` JS-output updates to use checked `res` fences, and migrate the docs away from ` ```res example `.

**Architecture:** Centralize fence classification in `scripts/test-examples.mjs` so page-level compile checks and `CodeTab` JS-output handling both rely on the same checked-fence contract. Checked plain `res` fences become the new default, `res nocheck` is skipped everywhere, and `CodeTab` updates continue to be `--update`-only while ignoring `TypeScript Output` tabs and preserving non-default labels beyond the first insertion case.

**Tech Stack:** Node.js 22 ESM, `node:test`, `tinyglobby`, ReScript compiler CLI via local `rescript` package, MDX docs under `markdown-pages/docs`

## Context

Today the checker only treats these as page-level compile inputs:

- ` ```res example `
- ` ```res prelude `
- ` ```res sig `

That same `example` requirement also drives `CodeTab` JS-output management. The approved contract changes that default:

- checked by default: ` ```res `
- explicit opt-out: ` ```res nocheck `
- unchanged special cases: ` ```res prelude ` and ` ```res sig `

The docs tree the checker currently scans is:

- `markdown-pages/docs/manual`
- `markdown-pages/docs/react`

Current ReScript-first `CodeTab` label shapes in that tree:

- `["ReScript", "JS Output"]`
- `["ReScript", "JS Output (Module)"]`
- `["ReScript"]`
- `["ReScript", "TypeScript Output"]`

Known manually invalid ReScript-first `CodeTab` examples that must become `nocheck` during migration:

- `markdown-pages/docs/manual/jsx.mdx`
  - the “Multiple spreads are not allowed” block
  - the “The spread must be at the first position” block
- `markdown-pages/docs/react/extensions-of-props.mdx`
  - the “Multiple spreads are not allowed” block
  - the “The spread must be at the first position” block

- [ ] **Step 1: Replace the old ad-hoc `res example` detection with shared fence classification helpers**

In `scripts/test-examples.mjs`, replace the current single-purpose `fenceKind` approach with a shared helper family near the existing `splitLines` helper:

````js
let classifyResFence = (line) => {
  if (line.startsWith("```res prelude")) {
    return "res-prelude";
  }

  if (line.startsWith("```res sig")) {
    return "res-sig";
  }

  if (line.startsWith("```res nocheck")) {
    return "res-nocheck";
  }

  if (line.startsWith("```res")) {
    return "res";
  }

  return null;
};

let classifyFence = (line) => {
  let resKind = classifyResFence(line);
  if (resKind != null) {
    return resKind;
  }

  if (line.startsWith("```js") || line.startsWith("```javascript")) {
    return "js";
  }

  if (line.startsWith("```ts") || line.startsWith("```typescript")) {
    return "ts";
  }

  return null;
};
````

This is the core contract change: plain ` ```res ` becomes checked by default, while `res nocheck`, `res prelude`, and `res sig` remain explicit variants.

- [ ] **Step 2: Rewrite `parseFile` so page-level compile input includes checked `res` fences and excludes `nocheck`**

Update `parseFile` so it wraps:

- `res` fences as example modules
- `res prelude` fences as includes
- `res sig` fences as module types

and completely ignores:

- `res nocheck`

The replacement shape should look like:

````js
let parseFile = (content) => {
  if (!/```res(?:\s|$)/.test(content)) {
    return;
  }

  let inWrappedBlock = false;
  let inIgnoredBlock = false;
  let moduleId = 0;

  return content
    .split("\n")
    .map((line) => {
      let kind = classifyResFence(line);

      if (kind === "res") {
        inWrappedBlock = true;
        return `/* _MODULE_EXAMPLE_START */ module M_${moduleId++} = {`;
      }

      if (kind === "res-prelude") {
        inWrappedBlock = true;
        return "/* _MODULE_PRELUDE_START */ include {";
      }

      if (kind === "res-sig") {
        inWrappedBlock = true;
        return `/* _MODULE_SIG_START */ module type M_${moduleId++} = {`;
      }

      if (kind === "res-nocheck") {
        inIgnoredBlock = true;
        return "";
      }

      if (line.startsWith("```")) {
        if (inWrappedBlock) {
          inWrappedBlock = false;
          return "} // _MODULE_END";
        }

        if (inIgnoredBlock) {
          inIgnoredBlock = false;
        }
      }

      if (inIgnoredBlock) {
        return "";
      }

      return inWrappedBlock ? line : "";
    })
    .join("\n");
};
````
